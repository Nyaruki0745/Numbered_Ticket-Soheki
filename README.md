# 文化祭 整理券システム — GitHub Pages 側

## ファイル構成

```
seiriken/
├── index.html          # スタッフ操作画面（登録・受付・統計）
├── monitor.html        # 呼出モニター（大型ディスプレイ用）
├── config.js           # ★ 全設定はここだけ編集する
├── css/
│   └── style.css       # 共通スタイル
└── js/
    └── api.js          # GAS通信モジュール
```
---

## 各画面の使い方

### index.html（スタッフ操作）

| タブ | 操作 |
|------|------|
| 登録 | 人数・開始/終了時間を入力して「整理券を発行」→ 番号表示 |
| 受付 | 整理番号を手入力 or QRスキャン → ステータス変更ボタン |
| 統計 | 残枠・ステータス別件数の確認（「更新」ボタンで再取得） |

ページ遷移なしでタブ切替のため、**入力状態は保持**されます。

### monitor.html（呼出モニター）

- 別ウィンドウ・別端末で開く（テレビやタブレット推奨）
- ログイン後、`POLL_INTERVAL_MS`（デフォルト15秒）ごとに自動更新
- 「呼出中」カードは黄色ハイライト＋点滅、「予約済」は青
- 上部のシートボタンで複数シートを切り替えて確認可能

---

## 整理番号の形式

```
A01-0001
^^^  ^^^^
│    └── 連番4桁（GASが自動採番）
└─────── 識別コード3桁（config.js の code で設定）
```

識別コードはシートと1対1で紐づくため、他クラスの券との混同を防げます。

---

## セキュリティ

- シート名（= スプレッドシートのシート名）は外部非公開
- 4桁パスワードはリクエストごとにGASへ送りサーバー側で検証
- GitHub Pages のソースを見られてもパスワードは config.js に平文で入っているため、
  **リポジトリは必ずプライベートにしてください**

---

## QRコード運用

`config.js` の `QR_MODE` で切り替えます。

| モード | 説明 |
|--------|------|
| `"number"` | QRの内容が整理番号そのもの（例: `A01-0001`）。スキャンで自動照合 |
| `"common"` | 共通QR確認のみ。番号は口頭または手入力で確認 |

先印刷チケットの場合、番号ごとのQRを印刷して `QR_MODE: "number"` を推奨します。
